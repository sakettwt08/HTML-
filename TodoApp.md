# Making PRD of TODO APP

## Objective 

- Build an HTML Website for Todo App. Make it offline accessible 

## Scope 

- Build a basic skeleton with tasks and a checkbox 

## Features And Requirements 

### P1 

- Show the tasks to be done 
- User can add new tasks and set the time to complete them
- Mark tasks as complete with checkbox

### P2 

- User should get notifications for tasks
- Edit existing tasks
- Delete completed tasks
- Multiple users can access and modify the same Todo list

## User Story

- The user will manage and keep their daily tasks in the list, which will help them in organizing their day-to-day work

## Technical Requirements 
- HTML 

## Design Requirements 

- Make the color theme dark and vibrant
- Make the tasks and checkboxes in dark color
- Use clear, readable fonts for better usability
- Add visual feedback for completed tasks

## Success Matrix

- Users can efficiently manage daily tasks
- Task completion rate increases by 40%
- User satisfaction rating above 4.5/5

## TimeLine

- 10 Days
